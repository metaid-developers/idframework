# Demo Note Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `demo-note/` as a full IDFramework-based on-chain notebook demo that is functionally compatible with `idnote-sample` and chain-data compatible with `/protocols/simplenote`.

**Architecture:** Keep `demo-note/` as a thin bootstrap layer and move reusable note-domain logic into `idframework/` through new note commands, Web Components, route utilities, upload helpers, and draft persistence helpers. Preserve IDFramework’s View/Command/Store split: components render and dispatch, commands own business logic, stores hold application state. Route synchronization is command-driven via `SyncNoteRouteCommand`; `demo-note/app.js` is the single bootstrap source of truth for store shape, i18n registration, command registration, and browser event wiring.

**Tech Stack:** Native ES modules, Web Components, Alpine stores, IDFramework command/delegate APIs, Node `node:test`, IndexedDB, Vditor vendor assets, existing MetaID wallet APIs.
